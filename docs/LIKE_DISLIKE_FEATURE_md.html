    Like/Dislike Recommendation Feature¶ ↑

    Overview¶ ↑

    This feature allows users to provide feedback on travel recommendations by liking or disliking them. The system learns from this feedback to provide better, more personalized recommendations in the future.

    Implementation Details¶ ↑

    1. Database Schema¶ ↑

    Migration: db/migrate/20250109000001_create_recommendation_feedbacks.rb

    Creates the recommendation_feedbacks table with: - user_id - Foreign key to users table - destination_city - The city that was recommended - destination_country - The country of the destination - trip_type - Type of trip (Solo, Couple, Family, Group) - travel_style - Travel style (Budget, Moderate, Luxury, etc.) - budget_min and budget_max - Budget range for the trip - length_of_stay - Duration in days - feedback_type - Either ‚Äòlike‚Äô or ‚Äòdislike‚Äô - reason - Optional text explaining why (max 500 chars) - Unique index on (user_id, destination_city, destination_country) - prevents duplicate feedback

    2. Model Layer¶ ↑

    File: app/models/recommendation_feedback.rb

    Validations: - Presence: user_id, destination_city, destination_country, feedback_type - Length limits: - city/country: 100 characters (prevents XSS/buffer attacks) - reason: 500 characters - Inclusion: feedback_type must be ‚Äòlike‚Äô or ‚Äòdislike‚Äô - Uniqueness: One feedback per user per destination

    Scopes: - likes - Returns all liked feedbacks - dislikes - Returns all disliked feedbacks - recent - Returns feedbacks ordered by most recent

    Class Methods: - user_preferences(user_id) - Returns hash with: - liked_destinations - Array of {city, country, travel_style, trip_type, length_of_stay} - disliked_destinations - Same structure as liked - preferred_styles - Travel styles the user tends to like - avoided_styles - Travel styles the user tends to dislike

    Association: - app/models/user.rb - Added has_many :recommendation_feedbacks, dependent: :destroy

    3. Controller Layer¶ ↑

    File: app/controllers/recommendation_feedbacks_controller.rb

    Actions:

    1. create (POST /recommendation_feedbacks)

    2. Accepts JSON feedback data

    3. Validates and sanitizes input (strips whitespace, enforces length limits)

    4. Updates existing feedback if user already rated this destination

    5. Creates new feedback otherwise

    6. Returns JSON: {success: true/false, message: string, feedback_type: string}

    7. destroy (DELETE /recommendation_feedbacks/:id)

    8. Removes feedback by ID

    9. Only allows users to delete their own feedback (security)

    10. Returns JSON: {success: true/false, message: string}

    11. index (GET /recommendation_feedbacks)

    12. Shows user's feedback history

    13. Groups by liked/disliked

    14. Limits to 50 most recent

    Security Features: - before_action :require_login - Authentication required - sanitize_feedback_params - Strips whitespace, enforces length limits - Authorization check - Users can only manage their own feedback - Strong parameters - Whitelist only allowed fields

    4. Routes¶ ↑

    File: config/routes.rb

    resources :recommendation_feedbacks, only: [:create, :destroy, :index]

    8. Usage Flow¶ ↑

    1. User generates recommendations

    2. Fills out travel preferences form

    3. Submits and receives AI recommendations

    4. User provides feedback

    5. Clicks "Like" or "Dislike" on a recommendation card

    6. Button changes color to show current state

    7. Feedback saved to database

    8. Success message appears

    9. User generates new recommendations

    10. System fetches user's feedback history

    11. AI receives personalized context

    12. AI suggests destinations similar to liked ones

    13. AI avoids destinations similar to disliked ones

    14. User views feedback history

    15. Visits /recommendation_feedbacks

    16. Sees all liked/disliked destinations

    17. Can remove feedback to reset preferences

    9. Testing the Feature¶ ↑

    Steps to verify:

    1. Run migration: bash bin/rails db:migrate

    2. Start server and login

    3. Generate travel recommendations

    4. Click "Like" on one recommendation

    5. Verify button turns green

    6. Check database: RecommendationFeedback.last

    7. Click "Dislike" on another recommendation

    8. Verify button turns red

    9. Generate new recommendations with same preferences

    10. AI should avoid disliked destinations

    11. AI should suggest similar to liked destinations

    12. Visit /recommendation_feedbacks

    13. Verify liked/disliked destinations appear

    14. Test "Remove" button

    15. Security tests:

    16. Try submitting feedback without login (should fail)

    17. Try deleting another user's feedback (should fail)

    18. Try XSS in reason field (should be sanitized)
